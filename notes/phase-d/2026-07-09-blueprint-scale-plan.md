# Phase D-5 蓝图规模化准备

**日期**: 2026-07-09
**阶段**: Phase D · 出题与判题 P0
**结论**: 先建立规模化计划和 dry-run 覆盖报告,不直接批量调用模型生成 140 首。

## 为什么不直接全量生成

蓝图规模化不是简单跑 140 次模型。题库是 demo 的可信度核心,如果直接把未审核蓝图和题目灌进数据库,会引入低质量题、重复考点和后续清理成本。

因此本轮做三件事:

- 明确当前覆盖口径。
- 估算全量规模和批次。
- 固定人工抽检方法。

## 新增脚本

`pnpm plan:blueprints`

只读数据库,不调用 LLM,不写入数据。输出:

- 当前诗词数、蓝图数、v2 题覆盖诗数。
- 未覆盖蓝图数量。
- 已有蓝图但缺 v2 题的诗。
- 按学段/体裁的覆盖结构。
- 第一批建议处理的 20 首。
- 10% 人工抽检样本。
- 执行步骤。

## 规模化口径

- 当前全库基础诗词: 140 首。
- 当前 v2 深做题库: 3 首 / 20 道。
- 当前蓝图表: 4 份,其中《春晓》是生成器测试数据,不等于已进入 demo 题库。
- 当前缺蓝图: 136 首。
- 已有蓝图但缺 v2 题: 1 首。
- 全量扩展剩余预估: 约 892 道 v2 题。
- 全量扩展时,绝句类约 6 个考点,律诗/长诗约 8 个考点。

## 第一批建议

`pnpm plan:blueprints` 当前建议优先处理中学/小学高频、常见考点主题和长诗,例如:

- 《长歌行》
- 《芙蓉楼送辛渐》
- 《送元二使安西》
- 《别董大》
- 《凉州词》
- 《送杜少府之任蜀州》
- 《黄鹤楼》
- 《闻官军收河南河北》
- 《望月怀远》

## 推荐流程

1. 每批 20 首生成蓝图。
2. 人工审核至少 10%,并额外审核长诗、低置信、核心 demo 候选诗。
3. 审核通过后导入蓝图。
4. 按蓝图生成 v2 题。
5. 跑 `pnpm audit:quiz`,critical 必须为 0。
6. 如果改 prompt,跑 `pnpm eval`。
