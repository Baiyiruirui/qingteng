# Phase E · 对话、Memory 与朗读加固

日期：2026-07-20

## 本轮目标

把已经可演示的功能补到更接近工程落地的状态：服务端不信任客户端历史，消息写入可幂等恢复，模型调用有明确预算，Memory 可由用户控制，朗读页同时支持听、看拼音和跟读。

## 对话可靠性

- 客户端只发送本轮消息与 `clientMessageId`，历史由服务端按用户和 conversation 所有权从 PostgreSQL 重建。
- 普通聊天与沉浸聊天共用 Redis 轮次锁、心跳、完成结果缓存和恢复队列。
- 重复中的请求返回冲突，已完成请求直接重放；持久化失败由后续请求重试。
- 日常聊天、沉浸开场和沉浸对话统一输出 token 上限、硬超时和 `req.signal` 中断传播。
- 高成本模型调用继续受用户/IP 预算约束；可重放结果在限流前识别，避免重复计费。

## Memory 用户控制

- 新增当前用户 Memory 列表、单条纠正、单条删除和全部清理 API。
- 新增跨设备暂停开关和 30/90/180 天或永久保留策略。
- 短期、中期、长期写入与抽取入口统一检查暂停状态；过期内容按策略惰性清理。
- UI 默认展示三条摘要，展开后进入完整管理，减少个人页信息密度。
- 删除和纠正始终按 owner scope 查询，不接受客户端指定其他用户。

## 儿童友好朗读

- 腾讯 TTS 提供标准示范音；一句过长时服务端分段。
- 云端字幕的完整音素序列转换为拼音，不完整时不展示错误拼音。
- 支持逐句跟读、整首挑战、当前句进度、3 秒准备倒计时和录音重试。
- 腾讯 ASR 结果只用于字句对齐，反馈口径为“朗读掌握度”，不伪装成声学发音评分。
- 桌面和 390px 移动端均完成截图走查；移动页面自然高度约 940px，拼音真实请求成功落态。

## 残余风险

- 当前幂等最终防线依赖 Redis；正式多实例生产应补数据库唯一约束。
- Memory 保留策略采用惰性清理，不是后台定时任务。
- 腾讯 TTS 拼音以完整字幕为门槛，服务不可用时回退为浏览器朗读且不显示云端拼音。
- 腾讯云和 Langfuse 凭据曾出现在协作截图中，发布前仍必须人工轮换并更新 Vercel。
