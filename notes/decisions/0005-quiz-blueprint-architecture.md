# 0005 · 为什么用考点蓝图驱动出题（而不是自由出题）

**日期**: 2026-06-22
**决策**: 引入"考点蓝图"层，每首诗预定义一组互斥考点，按蓝图出题

---

## 问题：v1 自由出题的同质化

v1 按"题型 × 难度"机械生成（4 种题型 × 3 难度 = 36 道），实测发现：

1. **扎堆名句**：LLM 自由选择时偏向最出名的诗句。《登高》12 道里有 7-8 道集中在颔联"无边落木萧萧下，不尽长江滚滚来"。
2. **考点重复**：同一考查能力换个问法反复出现（3 道翻译 3 道赏析都问同一联）。
3. **偏离考纲**：没有按照"鉴赏形象/鉴赏语言/鉴赏表达技巧/点评思想内容/点评作者观点"五大中考能力点分布。
4. **填空题挖单字**：v1 的 fill 型题往往挖 1 个字，不符合中考"理解性默写"的情境题范式。

---

## 解决方案：考点蓝图

每首诗人工设计一份蓝图（`data/quiz-blueprints.json`），定义：
- 几个互斥考点（绝句 4-6 个，律诗 6-8 个）
- 每个考点的类型、目标诗句、出题提示、参考答案方向
- 建议的题型（form）

出题时 LLM 按蓝图逐考点出题，每个考点 1 道题。

### 考点类型对应中考五大能力

| 考点类型 | 对应能力 |
|---|---|
| 默写 | 识记+理解 |
| 炼字 | 鉴赏语言 |
| 画面 | 鉴赏形象 |
| 意象 | 鉴赏形象 |
| 手法 | 鉴赏表达技巧 |
| 情感 | 点评思想内容 |
| 翻译 | 理解 |
| 综合选择 | 分析综合 |

---

## 为什么可控可扩展

### 人工设计样板定标准
3 首诗的蓝图由用户人工设计，确立了"什么是好的考点蓝图"的标准。

### LLM 按蓝图规模化生成
- 每个考点的 `prompt_hint` 和 `answerKey` 给了 LLM 明确的出题方向
- LLM 只需要在约束内把题目表达出来，不需要"自由发挥选择出什么"

### 蓝图生成器为扩展铺路
`src/ai/quiz/generate-blueprint.ts` 可以为任意诗生成蓝图：
输入：完整诗歌结构化数据 → 输出：考点蓝图 JSON

扩展路径：
1. 人工审核 AI 生成的蓝图（"人在回路"）
2. 批量铺到其余 137 首诗
3. 蓝图迭代改进（根据学生答题数据反馈调整考点分布）

---

## 取舍

**没有直接用 RAG 检索考点**：数据量少（140 首），RAG 的收益不如直接把整首诗结构化注入。

**没有全自动**：蓝图需要人工设计（或 AI 生成后人工审核）。这是有意的设计——AI 生成的蓝图可能遗漏重要考点或选择平庸考点，需要教研员把关。

**v1 保留**：36 道 v1 题作为对照组，可以直观看到"有蓝图/无蓝图"的出题质量差异。
