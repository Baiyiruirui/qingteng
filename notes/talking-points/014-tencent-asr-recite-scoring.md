# 014 · 腾讯 ASR 朗读评分

**适用场景**: 面试/Demo 被问"没有 OpenAI Whisper 怎么做朗读评分"时

---

## 一句话讲点

青藤的朗读评分不依赖 Whisper。当前版本用腾讯云 ASR 做中文语音转写,再用本地逐字对齐算法和原诗比较,输出掌握度、漏读字和青藤口吻反馈。

---

## 为什么改路线

原章程写的是 Whisper,但实际环境没有 OpenAI key。腾讯云 ASR 已有免费额度,并且国内网络可用,更适合作品集 demo 的稳定演示。

这个改动没有改变产品目标:

- 仍然是"录音 -> 转写 -> 评分 -> 反馈"
- 仍然写入 `events`,可以进入学习画像
- 只是把 STT provider 从 Whisper 换成腾讯 ASR

---

## 技术链路

1. `/recite/[poemId]` 展示原诗和录音 UI
2. 浏览器用 Web Audio API 采集麦克风
3. 客户端把音频降采样并编码成 16k WAV
4. `/api/recite/judge` 服务端用 TC3-HMAC-SHA256 签名调用腾讯 ASR `SentenceRecognition`
5. 识别结果进入 `scoreRecitation()`
6. LCS 对齐原诗汉字序列,算:
   - `accuracy`
   - `matchedChars`
   - `missingChars`
   - `extraChars`
7. 写入 `events(type='recite')`

---

## 为什么不用拼音相似度第一版

拼音相似度对多音字、儿化、轻声和 ASR 误识别比较敏感,需要一张诗词读音表。第一版先做字符级 LCS,优点是:

- 可解释
- 不依赖额外词典
- 适合 demo 验证闭环
- 对少量漏字/插字有容错

后续可以把字符级评分升级成"字形命中 + 拼音相似 + 停顿节奏"三层评分。

---

## 安全边界

- 腾讯密钥只读 `.env.local`
- 不在日志打印 `SecretId` / `SecretKey`
- 不把音频落盘,请求内 Base64 调用 ASR
- 只记录转写文本、得分和 requestId

---

## Demo 讲法

先从诗笺地图进入一首诗的"朗读",读一遍四句诗。提交后展示:

- ASR 识别到的文字
- 朗读掌握度
- 漏读/未对齐的字
- 青藤的鼓励式反馈

这条链路能证明我不是只会调 LLM,而是能把浏览器音频、云 ASR、算法评分、学习画像事件串成完整 AI 产品闭环。
