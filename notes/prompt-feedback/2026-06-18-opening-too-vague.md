# 回访开场白用泛化氛围代替具体回忆

**日期**: 2026-06-18
**Prompt 版本**: v1.0.0
**模型**: deepseek-chat

## 用户输入

> 用户上次对话内容涉及:《登高》、"萧萧"这个词、孤独感、情绪难受

## AI 实际回复

> 回来了?秋意正浓,台阶上该有片落叶了。

## 问题分析

开场白只捕捉到了"秋天/落叶"这一泛化氛围,没有提到任何具体内容:
- 没提《登高》这首诗
- 没提"萧萧"这个让用户有感触的词
- 没提用户表达过的孤独/难受情绪

根本原因:prompt 指令太软——"可以自然提到上次聊的话题"给了模型太大选择空间,模型选择了最含蓄、风险最低的表达(只说氛围,不说具体),看起来文艺但实际上没有"记住"的感觉。用户感受不到被记住,个性化 Memory 的价值就完全失去了。

## 期望回复

> 上次你说《登高》里"萧萧"两个字让你心里发闷——那种感觉还在吗?

或者:

> 你上次说背《登高》背得很难受,那首诗最后有没有背下来?

关键:必须有一个具体的钩子(诗名/词句/情绪),让用户一眼认出"青藤真的记住了"。

## 修复方案

**L1 · Prompt 修改**(`src/ai/prompts/v1/opening.ts` 回访分支):

原指令:
```
- 可以自然提到上次聊的话题或诗
- 或者关心一下他的状态
```

改为明确约束:
```
- 必须具体提到上次对话里的一个实际细节:可以是聊过的某首诗的名字、某个让他有感触的词句、或他表达过的某种情绪/困惑
- 不要用泛泛的氛围描写(比如只说"天凉了""秋意浓"这类)代替具体回忆
```

版本号从 v1.0.0 升到 v1.1.0。

核心原则:对生成式模型,**禁止项必须明确说出来**,不能指望它自己判断"什么算含蓄过头"。
