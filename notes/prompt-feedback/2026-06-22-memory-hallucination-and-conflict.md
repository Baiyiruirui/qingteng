# 长期记忆召回后的幻觉 + 多记忆冲突

**日期**: 2026-06-22
**Prompt 版本**: opening v1.1.0 / build-context v1.0.0
**模型**: deepseek-chat

## 问题 1:基于抽象记忆幻觉出虚假共同经历(致命)

### 现象

用户只聊过两条有意义的内容,被提炼为长期记忆:
- "这位学生偏爱李白,不喜欢格律严谨的律诗"(preference)
- "这位学生为《登高》的孤独感而动容"(emotion)

但青藤开场白说出了:**"今天更想聊聊李白折菊的事""上次聊到菊花"**

用户从未聊过李白折菊,也没有菊花相关的对话。这是完全的幻觉。

### 危害

- **直接破坏信任**:用户知道自己没说过这些,青藤"记错了"的感知比"没记住"更糟
- **对比短期 Memory 的准确**:反差越大,长期 Memory 的失败越明显
- **和产品核心价值背道而驰**:我们做记忆系统是为了"被自然记得",不是"被乱记"

### 根因

RAG grounding 缺失。模型拿到抽象记忆"喜欢李白"后,开场白 prompt 要求"必须提一个具体细节"——模型在没有具体细节可用的情况下,选择自由演绎出"李白折菊"作为"具体细节"。

本质上是 RAG 系统经典问题:**retrieval 给了 grounding,但 generation 没有被约束只能 ground 在 retrieved 内容里**。

### 修复

在 `opening.ts`(v1.1.0 → v1.2.0)和 `build-context.ts` 的 `renderMemoryContext` 里,明确加入:

```
⚠️ 绝对不得编造你们"上次聊过"但记忆里并未记录的具体事件
⚠️ 绝对不得把抽象印象(如"他喜欢李白")演绎成虚构的具体经历
你只能引用记忆中确实存在的内容。记忆只说"他喜欢李白",你就只知道这五个字。
```

在 opening.ts 里额外加:
```
你只能引用以上对话记录中字面上出现过的内容。对话里说了"李白",你就只知道"李白",不知道任何关于李白的具体故事。
```

## 问题 2:多记忆冲突时顾此失彼,违背用户明确偏好

### 现象

用户有两条记忆:
- preference: "这位学生偏爱李白,不喜欢格律严谨的律诗"
- emotion: "这位学生为《登高》的孤独感而动容"

新对话说"推荐一首诗",青藤召回了 emotion 记忆(情感更近),忽略了 preference 记忆(偏好较远),结果**推荐了刘禹锡的七律**——恰恰是用户明确讨厌的律诗类型。

### 根因

1. **召回层**:纯余弦相似度排序,emotion 记忆语义上和"推荐诗"更近(用了"感动""孤独"等情感词),preference 记忆相似度低,被 top-3 截断
2. **注入层**:记忆被混在一起列出,模型没有被提醒 preference 是硬约束

### 修复

**召回层**(`long-term.ts`):两阶段召回
- preference 类记忆:阈值降低到 0.15(几乎总是召回),保证偏好不被丢失
- 其他类型:标准阈值 0.4
- 合并时 preference 排在前面

**注入层**(`build-context.ts`):分组渲染 + 硬约束提示
- 偏好单独一组,标注"选择诗歌时务必尊重,这是硬约束"
- 情绪单独一组,标注"影响语气,不影响选诗偏好"
- 末尾加:"推荐诗歌时,既要照顾他的情绪,也不要违背他的明确偏好"
